# Release-Review Fixture JSON Schema

Release-review fixtures under `fixtures/release-review/current/` are checked
JSON examples of deterministic report output.

The fixture files currently map to report commands:

| Fixture | Source command | Schema status |
|---|---|---|
| `golden-inventory.json` | `verityfoundry report golden-inventory --format json` | documented by report tests |
| `matrix-coverage.json` | `verityfoundry report matrix-coverage --format json` | documented by report tests |
| `portfolio-coverage.json` | `verityfoundry report portfolio-coverage --format json` | `schemas/portfolio-coverage-report.schema.json` |
| `provenance-distribution.json` | `verityfoundry report provenance-distribution --format json` | `schemas/provenance-distribution-report.schema.json` |

## Current Contract

The fixture contract is:

1. A fixture must be generated by a deterministic local report command.
2. The fixture must not require an external AI API.
3. The fixture must be tested against current report output.
4. The fixture must be included in packaged data.
5. The fixture must not be described as VeritySpec validation evidence,
   readiness approval, or human approval.

The active fixture equality test is:

```bash
python -m unittest tests.test_release_review_fixtures -v
```

The active schema and snapshot test is:

```bash
python -m unittest tests.test_release_review_snapshots -v
```

## Adding a New Fixture

When adding a release-review fixture:

1. Add or confirm the deterministic report command.
2. Generate the fixture under `fixtures/release-review/current/`.
3. Add a report-specific JSON schema when downstream tooling needs stable
   fields.
4. Add a test that compares the fixture to current report output.
5. Add package-data coverage or confirm existing package-data globs include
   the fixture.
6. Link docs from `README.md`.
7. Update `CHANGELOG.md` and `ROADMAP.md`.

## Authority Boundary

Release-review fixtures describe VerityFoundry repository state. They do not
prove that a candidate VeritySpec workspace is valid, complete, approved,
release-ready, compliant, or implementation-ready.

For product-contract authority, run VeritySpec validation, linting, graph, and
readiness checks after converting candidate fixtures into real workspaces.
