# Unity Game Workflows

Unity game workflows turn rough creative input into candidate VeritySpec
workspace drafts.

Inputs may include:

- game name
- short description
- rough GDD
- concept art notes
- identity image notes
- target platforms
- Unity version
- shared Unity runtime assumptions
- prototype goals
- liveops assumptions

Candidate outputs may include:

- game product records
- GDD source records
- visual identity records
- concept art records
- core loop records
- game mode records
- feature records
- Unity dependency records
- telemetry intent records
- readiness gaps
- agent implementation context

Use `verityfoundry matrix unity-game` to inspect the initial matrix.
