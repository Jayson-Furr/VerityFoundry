# Unity Shared Library Workflows

Unity shared-library workflows draft candidate VeritySpec records for reusable
Unity runtime packages and shared systems.

Inputs may include:

- library name
- README
- Unity package notes
- API notes
- shared systems
- target Unity versions
- consumer games
- operations expectations
- archive expectations

Candidate outputs may include:

- library product record outline
- Unity package records
- capability records
- dependency records
- consumer-facing contract notes
- version compatibility assumptions
- readiness gaps

Future VeritySpec workspace dependency support should make this workflow more
powerful, but current prompts must mark cross-workspace references as
assumptions.
