# Candidate Workspace Outline

The dependency map should create a candidate portfolio/integration workspace
that records dependency assumptions without claiming VeritySpec dependency
resolution already exists.

Expected records:

- dependency-map product identity
- one dependency record per game consumer
- illustrative cross-workspace reference candidates
- exported-record assumptions
- dependency visibility and lockfile risks
- readiness gap record

Friendly references such as `sharedUnity::unity.package.save_system` are
illustrative authoring syntax only. VeritySpec remains the authority for future
dependency resolution, exported-record validation, lockfiles, and canonical
URI forms.
